# 가이드 — [Auth] 커뮤니티 앱

> 로그인한 사용자만 글을 쓰고, 자기 글만 수정/삭제하는 커뮤니티 앱.

## 전제
- 튜토리얼 Sign In & Sign Up & Profile, Todo App(+Login & DB) 완료. Supabase 프로젝트 준비.

## 미션
- **Part 1 회원가입 & 로그인** (Supabase Auth)
- **Part 2 게시글 CRUD**: 작성=로그인 사용자만(제목+내용), 조회=로그인 누구나(작성자 표시), 수정/삭제=본인 글만
- **Part 3 게시글 목록**: 최신순(제목/작성자/시간), 클릭 시 상세
- **Part 4 배포 & 공유**: Vercel 배포, 타인이 실제 가입·글쓰기

## 핵심 구조
```
[회원가입/로그인] → [게시글 목록(전체 공개)] → [글쓰기(로그인 필수)] → [수정/삭제(본인만)]
```
- Auth: 누가 썼는지 기록 · DB: 게시글+작성자 · 앱: 권한 분기

## 제출물
- 배포 URL(Vercel), GitHub 링크, 스크린샷(가입→로그인→글쓰기→목록), (보너스) 타인 가입·작성

## 팁
- Todo App 구조 재활용(Todo→게시글). Supabase RLS로 "본인 글만 수정/삭제". 디자인보다 Auth+CRUD 집중.
