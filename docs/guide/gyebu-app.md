# 가이드 — [Server+DB] 가계부 앱 (커리큘럼 원문)

> 하네스가 매 단계 `@docs/guide/gyebu-app.md` 로 주입하는 "반드시 따를" 원문 가이드.

> 💡 **Server + DB (Supabase)**를 조합해서 나만의 가계부 앱을 만드세요!

## 미션
1. 사용자가 **수입/지출 내역을 등록** (날짜, 금액, 카테고리, 메모)
2. 등록된 내역을 **목록으로 조회**
3. **카테고리별 합계** 표시 (식비, 교통, 주거, 구독료, 경조사 등)
4. 모든 데이터는 **DB (Supabase)에 저장**

## 핵심 구조
```
[사용자 입력(금액·카테고리·메모)] → [Server] → DB 저장 → 조회 & 카테고리별 합계 → 응답
```
- DB: 수입/지출 저장소 + 카테고리별 통계
- Server: CRUD API (등록/조회/수정/삭제)

## 제출물
- GitHub 저장소 링크(코드 포함)
- 동작 스크린샷(최소 1장)
- 에이전트와의 대화 스크린샷(최소 1장)

## 팁
- Supabase 테이블 컬럼: `type`(수입/지출), `category`, `amount`, `memo`, `date`
- 카테고리별 합계는 SQL `GROUP BY`
- 여유 시 월별 차트·예산 대비 사용량

## 포인트 획득 기준 (25pt)
| 항목 | 포인트 | 기준 |
|------|--------|------|
| 기본 완료 | 10pt | 제출물을 기한 내 제출 |
| 에이전트 활용 | 5pt | 에이전트와 2회 이상 대화하며 결과물을 개선한 흔적 |
| 창의성 | 5pt | 자기만의 개성이 담긴 결과물 (월별 리포트, 차트 시각화, 예산 대비 알림 등) |
| 공유 보너스 | 5pt | 단톡방에 결과물을 공유하고 다른 수강생 결과물에 리액션 |

- 달성 수준: 기본 제출 **10pt** / 기본 + 활용·창의성 **15~20pt** / 올클리어 **25pt**
