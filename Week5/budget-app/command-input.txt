Server + DB (Supabase)를 조합해서 나만의 가계부 앱을 만들어줘
1. 사용자가 수입/지출 내역을 등록할 수 있게 하세요 (날짜, 금액, 카테고리, 메모)
2. 등록된 내역을 목록으로 조회할 수 있게 하세요
3. 카테고리별 합계를 보여주세요 (식비, 교통, 주거, 구독료, 경조사 등)
4. 모든 데이터는 DB (Supabase)에 저장하세요

[사용자 입력 (금액, 카테고리, 메모)] → [Server] → DB에 수입/지출 저장 → 내역 조회 & 카테고리별 합계 계산 → 결과 응답
- DB 역할: 수입/지출 내역 저장소 + 카테고리별 통계 조회
- Server 역할: CRUD API 제공 (등록, 조회, 수정, 삭제)
- Supabase 테이블 컬럼: type(수입/지출), category(식비/교통/주거/구독료/경조사 등), amount, memo, date
- 카테고리별 합계는 SQL GROUP BY 활용
- 여유가 되면 월별 지출 차트나 예산 대비 사용량 표시에 도전
