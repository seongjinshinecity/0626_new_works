# 대화형 실행 가이드 (스크립트 없이)

`bash harness/run-loop.sh ...` 같은 명령을 몰라도 됩니다.
**Claude Code를 열고 → 아래 프롬프트를 붙여넣기** 만 하면, 개발→검증→수정 루프가 대화로 진행됩니다.

---

## 처음 한 번만: Claude Code 열기

1. **터미널(명령 프롬프트)** 을 연다.
   - 맥: `Cmd + Space` → "터미널" 입력 → 엔터
   - 윈도우: 시작 → "cmd" 또는 "PowerShell"
2. 개발할 프로젝트 폴더로 이동 (폴더를 드래그하면 경로가 들어갑니다):
   ```
   cd "여기에 프로젝트 폴더 경로"
   ```
3. Claude Code 실행:
   ```
   claude
   ```
   → 이제 채팅처럼 대화하면 됩니다. (스크립트 실행 아님!)

> 준비물: 이 폴더에 `.claude/agents/`, `specs/`, `docs/guide/` 가 복사돼 있고, `.env` 에 Supabase 값이 채워져 있어야 합니다. (안 돼 있으면 Claude가 "무엇이 필요한지" 멈추고 물어봅니다.)

---

## 가계부 앱 만들기 — 이 한 덩어리를 붙여넣으세요

```
가계부 앱(gyebu-app)을 개발할 거야. 다음 규칙을 지켜줘.

1. 먼저 @specs/gyebu-app.spec.md 와 @docs/guide/gyebu-app.md 를 끝까지 읽어.
   이 둘이 유일한 기준이야. 스펙에 없는 기능은 만들지 마.
2. 그다음 아래를 반복해:
   (1) builder 에이전트로 아직 통과 못 한 인수기준(AC)을 구현하고 'npm run build'를 통과시켜.
   (2) verifier 에이전트로 모든 AC를 실제로 실행해서 검증하고,
       AC별 통과/실패 표와 마지막에 'VERDICT: PASS' 또는 'VERDICT: FAIL'을 보여줘.
   (3) FAIL이면 실패 이유를 builder에게 주고 다시 (1)부터. PASS면 끝.
3. 최대 5번까지만 반복하고, 안 되면 멈춰서 뭐가 막혔는지 나한테 쉬운 말로 알려줘.
4. 중간에 Supabase 키처럼 내가 줘야 할 게 있으면 멈추고 물어봐.

지금 1번부터 시작해.
```

붙여넣고 엔터 → Claude가 알아서 만들고, 테스트하고, 실패하면 고치는 걸 **눈으로 보면서** 진행됩니다.

---

## 진행 중에 이렇게 말하면 됩니다 (자유롭게 대화)

- 멈췄을 때: `계속해`
- 다시 보고 싶을 때: `지금 통과한 AC랑 안 된 AC 다시 정리해줘`
- 특정 부분만: `AC4 카테고리 합계가 틀린 것 같아, 거기만 다시 확인해줘`
- 권한 물어보면: 파일 수정/명령 실행은 `허용`(또는 Yes) 눌러주면 됩니다.

## 다른 프로젝트도 똑같이 — 이름만 바꾸세요

위 프롬프트에서 **`gyebu-app` 세 군데**(스펙 경로 / 가이드 경로 / 첫 줄 이름)만 바꾸면 됩니다.

| 만들 것 | 바꿀 이름 |
|---|---|
| 커뮤니티 앱 | `community-app` |
| 쇼핑몰 | `shopping-mall` |
| 카페 대시보드(보스) | `cafe-dashboard` |
| 가계부 분석 에이전트 | `gyebu-agent` |
| 내 카페 아는 에이전트 | `my-cafe-agent` |
| 자동 리서치 스킬 | `research-skill` |
| 리뷰·경쟁사 보고서 | `review-report` |
| 인스타 인플루언서 | `instagram-influencer` |

> 단, gyebu-app 외에는 `specs/<이름>.spec.md` 가 아직 없습니다.
> 그 경우 프롬프트 1번 앞에 이 줄을 먼저 추가하면 Claude가 스펙부터 만들어 줍니다:
> `먼저 @docs/guide/<이름>.md 를 읽고 @specs/gyebu-app.spec.md 와 같은 형식으로 specs/<이름>.spec.md 를 만들어줘. 그다음 아래 루프를 진행해.`

## 한눈 요약
- 스크립트(`bash ...`) = 자동·한 번에 → 초보에겐 안 보여서 어려움
- **대화형 = `claude` 열고 프롬프트 붙여넣기** → 보면서, 말로 고치면서 진행 ✅
