# 전체 자동 실행 (1~9번 한 번에)

프로젝트 1번부터 9번까지 **순서대로 자동으로** 만들고 검증합니다. 명령은 딱 한 줄.

---

## 준비 (처음 한 번)

1. `loop-dev-setup` 폴더 안에서 환경변수 채우기:
   ```
   cp .env.sample .env
   ```
   → `.env` 파일을 열어 Supabase 값 등을 채웁니다.
2. MCP 설치 (한 번만, `스킬_MCP_설치안내.md` 참고):
   ```
   claude mcp add supabase --scope user --env SUPABASE_ACCESS_TOKEN=$SUPABASE_ACCESS_TOKEN -- npx -y @supabase/mcp-server-supabase@latest --project-ref=$SUPABASE_PROJECT_REF
   ```

## 실행 (이 한 줄)

`loop-dev-setup` 폴더에서:
```
bash harness/run-all.sh
```

끝입니다. 그러면 순서대로:
1. gyebu-app → 2. community-app → 3. shopping-mall → 4. cafe-dashboard
→ 5. gyebu-agent → 6. my-cafe-agent → 8. review-report 까지 **자동**으로 만들고 검증합니다.

- 각 프로젝트는 `build/<이름>/` 폴더에 생성됩니다.
- 한 프로젝트가 통과(VERDICT: PASS)하면 자동으로 다음으로 넘어갑니다.
- 마지막에 1~9번 **전체 성공/실패 요약표**를 보여줍니다.

반복 횟수를 바꾸려면:
```
bash harness/run-all.sh ./build 8     # 각 프로젝트 최대 8회까지 시도
```

## 자동이 안 되는 2개 (7·9번) — 그 자리에서 안내됨

- **7. research-skill** (스킬 재호출 검증)
- **9. instagram-influencer** (인스타에 사람이 직접 로그인 필요)

이 둘은 사람이 개입해야 해서, 실행기가 **자동으로 건너뛰면서 "이 폴더에서 claude 열고 이 프롬프트 붙여넣으세요"** 안내를 화면에 띄웁니다. 안내대로 따라 하면 됩니다. (`대화형_실행가이드.md` 와 동일한 방식)

---

## 진행 중 알아두면 좋은 것

- 중간에 권한을 물으면 허용해 주세요. 완전 무인으로 돌리려면 끝에 `--dangerously-skip-permissions` 를 붙일 수 있지만, **처음에는 권장하지 않습니다**(무엇을 하는지 보면서 진행하는 게 안전).
- 시간이 꽤 걸리고 API 사용량도 늘어납니다. 먼저 **1번(gyebu-app)만** 잘 도는지 보고 전체를 돌리는 걸 추천합니다:
  ```
  bash harness/run-loop.sh gyebu-app 5     # 1번만 테스트
  ```
- 어딘가 막혀 5회 안에 통과 못 하면, 그 프로젝트는 "에스컬레이션"으로 표시되고 다음으로 넘어갑니다. 로그는 `build/<이름>/harness/logs/` 에 남습니다.

## 요약
