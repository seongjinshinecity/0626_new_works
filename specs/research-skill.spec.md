# 자동 리서치 스킬 (프로젝트 7) — 스펙

## 목표
"리서치/조사/트렌드 정리" 류의 요청이 오면 **매번 같은 절차·같은 포맷**으로 웹 조사를 수행하는 **재사용 스킬**을 만든다. 일회성 답변이 아니라 `.claude/skills/research/SKILL.md`로 박제해 어느 대화에서나 동일하게 재현된다.

## 산출물
- `.claude/skills/research/SKILL.md` — 스킬 정의(언제 발동 + 절차 + 고정 출력 포맷).
- 실제 리서치 결과 `.md` 2개(다른 키워드, 동일 포맷) — `out/`.
- README.

## 스킬 절차 (SKILL.md에 박제)
1. 주제/키워드 확정
2. 수집: `WebSearch`로 검색 → 서로 다른 출처 **3곳 이상**, 필요 시 `WebFetch`로 본문 심화
3. 정리: 고정 템플릿(제목·수집일·키워드·출처별 핵심·한 줄 요약·다음 액션)으로 `.md` 작성, 출처는 링크
4. 공유(보너스): Notion MCP 있으면 페이지화, **없으면 건너뜀**(없는 연결을 지어내지 않음)

## 변별 검증 (이 프로젝트의 핵심)
- **메커니즘**: `WebSearch`/`WebFetch`가 실제 결과를 반환하는지 먼저 확인(headless에서 Browser MCP보다 안정적).
- **트리거**: description이 자연어 요청("리서치 해줘")에 실제로 발동하는지 — `claude -p`로 재호출해 확인.
- **재현성**: 키워드를 바꿔 **2회** 실행했을 때 섹션 구조가 동일한지(제목/메타/출처별 핵심/한 줄 요약/다음 액션).
- **사실성**: 출처 3곳 이상 + 구체 사실(숫자·고유명사) 인용. 지어낸 수치 금지.

## 제약 / 솔직한 한계
- Notion MCP **미연결** → 4단계는 파일 저장으로 대체(README에 명시).
- API 키 불필요(WebSearch/WebFetch는 키 없이 동작).
