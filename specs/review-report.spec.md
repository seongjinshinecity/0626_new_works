# 리뷰/경쟁 리포트 자동화 (프로젝트 8) — 스펙

## 목표
원천 텍스트 데이터(고객 리뷰 / 경쟁사 메모)를 **사람이 그대로 보고받을 수 있는 산출물**(.xlsx / .pptx)로 자동 변환한다. "분석을 글로 설명"이 아니라 **실제 파일 객체(차트·피벗·표·슬라이드)**를 만든다.

## 경로 A — 고객 리뷰(VoC) → 엑셀
- 입력: `cafe_reviews.csv` (20건, 컬럼: date, platform, rating, review, theme)
- 산출: `out/voc_analysis.xlsx` (5시트)
  1. 요약 — 핵심 인사이트(평균 별점, 부정 1위 테마)
  2. 리뷰원본 — 필터/고정행, 부정(★≤2) 행 하이라이트
  3. 별점분포 — 표 + **막대 차트 + 원형 차트**
  4. 테마피벗 — 전체 vs 부정 교차 + **막대 차트**
  5. 부정Top3 — 부정 테마 Top3 + 대표 리뷰 + 개선 액션
- 검증 포인트: 데이터에서 **부정 1위 = '대기시간'**이 자연 도출(하드코딩 금지). my_cafe.md의 '주말 피크 대기'와 일치.

## 경로 B — 경쟁사 메모 → PPT
- 입력: `competitors.md` (자사=데일리브루 + 경쟁 3사)
- 산출: `out/competitor_analysis.pptx` (5슬라이드)
  1. 표지 / 2. 시장개요 / 3. 비교표(실제 table 객체) / 4. 차별화 / 5. 추천 액션
- 스토리 연결: VoC 대기시간 pain → 차별화(예약/회전) → 추천(인플루언서 협업, 프로젝트 9 연계)

## 재현/검증 (이번 프로젝트의 변별 기준)
파일이 "써졌다"가 아니라 **열어서 객체가 실제 존재**해야 통과:
- xlsx: `load_workbook` 후 시트 5개 + `ws._charts` 합계 3개 확인
- pptx: `Presentation` 후 슬라이드 5개 + slide3 `has_table` 확인

## 제약
- LLM/외부 API 키 불필요 — 순수 openpyxl / python-pptx.
- 결론을 코드에 하드코딩하지 않고 데이터 집계에서 도출.
